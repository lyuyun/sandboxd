# Use Case × 特性能力映射

**版本** v1.1 · **日期** 2026-06-23  
**说明** 本文档列出每个 Use Case 所要求的 infra 特性集合，并对比 e2b 与 kuasar-sandbox 的覆盖状态。  
特性编号来源于 kuasar-sandbox 特性列表，状态标注：✅ 已支持 / 🔶 部分支持 / ❌ 缺失。

---

## 阅读说明

每个 Use Case 下的特性按**功能域**分组，格式如下：

```
[域名]
  特性 A    e2b ✅   kuasar-sandbox ✅    — 简要说明
  特性 B    e2b ✅   kuasar-sandbox ❌    — 简要说明（缺失原因）
```

每节末尾分别给出两个产品的覆盖度评估与可用性结论。  
文末附**横向汇总矩阵**和**e2b 与 kuasar-sandbox 可用性对比总表**。

## UC-A：代码执行（AI 驱动）

### UC-A1：交互式代码解释器

> 用户向 AI 提问 → AI 生成代码 → 沙箱执行 → 输出实时返回。

| 域 | 特性 | e2b | kuasar-sandbox | 说明 |
|---|---|---|---|---|
| **生命周期** | 快照恢复启动（snp） | ✅ | ✅ | 毫秒级恢复的基础 |
| | 自动恢复（数据面触发） | ✅ | ✅ | 沙箱 TTL 期间自动唤醒 |
| | TTL 刷新（KeepAlive） | ✅ | ✅ | 多轮对话保持会话 |
| | 自动暂停（TTL 到期） | ✅ | ✅ | 对话停止后释放资源 |
| | 恢复超时配置（park_timeout） | ✅ | ✅ | 控制等待窗口 |
| **运行时数据面** | 启动进程（任意命令） | ✅ | ✅ | 执行代码的核心 |
| | stdin/stdout/stderr 流式返回 | ✅ | ✅ | 实时输出反馈 |
| | PTY 终端支持 | ✅ | ✅ | 交互式 Shell |
| | 文件下载 | ✅ | ✅ | 取走执行结果（图表、数据文件）|
| | 进程 cgroup 管理（freeze/thaw） | ✅ | ✅ | 执行超时保护 |
| **内存管理** | UFFD 按需内存加载 | ✅ | ✅ | 快照恢复时无需全量加载内存 |
| | 内存超分（60×） | ❌ | ✅ | e2b 无此密度；kuasar-sandbox 低待机成本 |
| **代理层** | HTTP 透明代理 | ✅ | ✅ | 访问沙箱内服务 |
| | WebSocket 透明代理 | ✅ | ✅ | 流式输出类界面 |
| **安全** | 多租户数据隔离 | ✅ | ✅ | 不同用户的解释器完全隔离 |
| **可观测性** | 沙箱启动延迟指标 | ✅ | 🔶 | e2b 完整暴露；kuasar-sandbox 有内部统计未暴露 API |

**kuasar-sandbox 覆盖度**：核心路径全覆盖，缺失项不影响基本功能。**可用性：✅ 完整可用。**

**kuasar-sandbox 覆盖度**：扩缩容和迁移核心能力完整，且 Maglev 缓存亲和是技术优势；滚动升级和可观测性是运维体验缺口。**可用性：✅ 基本可用（无滚动升级）。**

> 执行用户/AI 生成的不可信代码，防止逃逸到宿主或其他沙箱。

| **安全** | 多租户数据隔离 | ✅ | ✅ | |

**e2b 覆盖度**：KVM 隔离 + Secure 模式 + 持久审计日志。**可用性：✅ 完整可用。**

**kuasar-sandbox 覆盖度**：核心 KVM 隔离优秀，eBPF 数据面持久化是额外安全加固；Secure 模式缺失是高安全场景短板。**可用性：✅ 基本可用（Secure 模式缺失）。**

### UC-H3：合规审计

> 受监管行业要求完整、持久、不可篡改的控制面操作审计记录。

| 域 | 特性 | e2b | kuasar-sandbox | 说明 |
|---|---|---|---|---|

| | 结构化日志聚合（SIEM 兼容）| ✅ | ❌ | e2b 可导出到 Splunk/Elasticsearch |
| | 审计日志含操作者/时间戳/结果 | ✅ | 🔶 | kuasar-sandbox 字段未完整规格化 |
| **安全** | AES-256-GCM 加密存储 | ✅ | ✅ | 密钥存储加密 |
| **运营** | Admin 操作记录 | ✅ | ✅ | |

**e2b 覆盖度**：持久审计日志 + SIEM 集成完整。**可用性：✅ 完整可用。**

**kuasar-sandbox 覆盖度**：审计记录本身有实现，但强制写 tmpfs 使其在合规场景无效。**可用性：❌ 不可用（合规阻断，但实现成本低——加 syslog forward 即可）。**

### UC-H4：数据静态保护

> 用户数据在存储和快照中加密，密钥用户控制，介质销毁安全。

| 域 | 特性 | e2b | kuasar-sandbox | 说明 |
|---|---|---|---|---|
| **块存储** | 收敛加密（相同内容→相同密文）| ❌ | ✅ | kuasar-sandbox 独有；跨租户安全去重 |
| | 密钥轮换（keytag）| ❌ | ✅ | kuasar-sandbox 独有 |
| **安全** | AES-256-GCM 密钥加密存储 | ✅ | ✅ | 密钥本身加密 |
| | 集群密钥 mTLS 分发 | ❌ | ✅ | kuasar-sandbox 独有传输加密机制 |
| | 标准快照加密存储 | ✅ | 🔶 | e2b 有标准块加密；kuasar-sandbox 用收敛加密 |
| **运行时** | 运行时写层静态加密（overlay）| ❌ | ❌ | **双方均缺失**，写层明文 |

**e2b 覆盖度**：标准快照加密已有；收敛加密、keytag 轮换、写层加密均缺失。**可用性：🔶 部分可用（写层未加密，无 BYOK）。**

**kuasar-sandbox 覆盖度**：收敛加密和 keytag 轮换是技术优势；运行时写层加密是最大缺口，双方均未实现。**可用性：🔶 部分可用（快照加密✅，写层加密❌）。**
